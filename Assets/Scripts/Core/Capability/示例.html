Core.Capability 框架开发指南

本指南将详细介绍 Core.Capability 这套基于实体-组件-能力（类似ECS）的框架，并带领你从零开始完成一个“移动、染色与攻击”的具体场景实现。

一、 这套系统有哪些组件，如何使用？

Core.Capability 是一套彻底去耦、纯运行时驱动的框架，其核心由以下四大主要部分构成：

1. CapabilityWorld (能力世界)

作用： 整个框架的入口和管理容器。它负责初始化能力系统、管理实体的生命周期，以及桥接外部的 Update/FixedUpdate 循环。

使用： 继承 CapabilityWorld，在 OnInitialize 中分配容量，并在外部主循环中调用其 OnUpdate。

2. CEntity (实体)

作用： 组件的容器。本身不包含任何业务逻辑，仅作为 Id 的载体，内部使用稀疏索引数组 (IndexedObjectArray) 挂载组件。

使用： 通过 world.AddChild() 创建，使用 AddComponent<T>() 和 RemoveComponent(id) 管理组件。

3. CComponent (组件)

作用： 纯数据载体。继承自 CComponent，用于描述实体的各种状态或属性。

4. CapabilityBase (能力/系统)

作用： 业务逻辑的执行者。监听特定组件的变化，当满足条件时激活 (ShouldActivate)，并在激活状态下每帧执行逻辑 (TickActive)。

使用： 继承 CapabilityBase，重写 OnInit (调用 Filter 注册监听)、ShouldActivate、ShouldDeactivate 以及 TickActive。

二、 具体场景示例：移动、染色与攻击

需求： 一个人 (Entity) 移动向敌人，在移动经过的格子上染上自己的势力颜色，并在接触后对敌人造成伤害。

1. 定义组件 (Components)

using Core.Capability;
using UnityEngine;

// 位置组件
public class PositionComponent : CComponent
{
    public Vector2Int CurrentGrid { get; set; }
    public Vector3 WorldPosition { get; set; }
}

// 移动目标组件
public class MoveTargetComponent : CComponent
{
    public Vector3 TargetPosition { get; set; }
    public float Speed { get; set; }
}

// 势力组件（用于染色）
public class FactionComponent : CComponent
{
    public Color FactionColor { get; set; }
}

// 生命值组件
public class HealthComponent : CComponent
{
    public float HP { get; set; }
}

// 攻击指令组件
public class AttackCommandComponent : CComponent
{
    public CEntity Target { get; set; }
    public float Damage { get; set; }
}

2. 定义能力 (Capabilities)

我们需要两个能力：一个处理移动与染色，一个处理攻击。

using Core.Capability;
using UnityEngine;

// === 移动与染色能力 ===
public class MovementAndColorCapability : CapabilityBase
{
    protected override void OnInit()
    {
        // 只有同时拥有 Position 和 MoveTarget 的实体才会触发评估
        Filter(ComponentId<PositionComponent>.TId, ComponentId<MoveTargetComponent>.TId);
    }

    public override bool ShouldActivate()
    {
        // 激活条件：拥有 Position 和 MoveTarget
        return Owner.HasComponent(ComponentId<PositionComponent>.TId) && 
               Owner.HasComponent(ComponentId<MoveTargetComponent>.TId);
    }

    public override bool ShouldDeactivate()
    {
        // 停用条件：失去 MoveTarget 时停止移动
        return !Owner.HasComponent(ComponentId<MoveTargetComponent>.TId);
    }

    public override void TickActive(float deltaTime, float realElapsedSeconds)
    {
        var pos = (PositionComponent)Owner.GetComponent(ComponentId<PositionComponent>.TId);
        var move = (MoveTargetComponent)Owner.GetComponent(ComponentId<MoveTargetComponent>.TId);

        // 执行移动逻辑
        Vector3 dir = (move.TargetPosition - pos.WorldPosition).normalized;
        pos.WorldPosition += dir * move.Speed * deltaTime;

        // 计算当前所在格子 (简化版逻辑)
        Vector2Int gridPos = new Vector2Int(Mathf.RoundToInt(pos.WorldPosition.x), Mathf.RoundToInt(pos.WorldPosition.z));
        
        // 如果跨越了格子，进行染色
        if (gridPos != pos.CurrentGrid)
        {
            pos.CurrentGrid = gridPos;
            var faction = (FactionComponent)Owner.GetComponent(ComponentId<FactionComponent>.TId);
            if (faction != null)
            {
                // TODO: 对接外部地图管理器进行实际网格染色
                Debug.Log($"将格子 {gridPos} 染成了 {faction.FactionColor}");
            }
        }

        // 到达目标后移除移动指令组件
        if (Vector3.Distance(pos.WorldPosition, move.TargetPosition) < 0.1f)
        {
            Owner.RemoveComponent(ComponentId<MoveTargetComponent>.TId);
            Debug.Log("到达目标");
        }
    }
}

// === 攻击能力 ===
public class AttackCapability : CapabilityBase
{
    protected override void OnInit()
    {
        // 监听攻击指令
        Filter(ComponentId<AttackCommandComponent>.TId);
    }

    public override bool ShouldActivate()
    {
        return Owner.HasComponent(ComponentId<AttackCommandComponent>.TId);
    }

    public override bool ShouldDeactivate()
    {
        return !Owner.HasComponent(ComponentId<AttackCommandComponent>.TId);
    }

    public override void TickActive(float deltaTime, float realElapsedSeconds)
    {
        var attack = (AttackCommandComponent)Owner.GetComponent(ComponentId<AttackCommandComponent>.TId);
        
        if (attack.Target != null && attack.Target.IsActive)
        {
            var targetHealth = (HealthComponent)attack.Target.GetComponent(ComponentId<HealthComponent>.TId);
            if (targetHealth != null)
            {
                targetHealth.HP -= attack.Damage;
                Debug.Log($"对目标造成 {attack.Damage} 点伤害，目标剩余血量: {targetHealth.HP}");
                
                // 血量归零，给目标挂载销毁组件
                if (targetHealth.HP <= 0)
                {
                    attack.Target.AddComponent<DestroyComponent>();
                }
            }
        }

        // 攻击动作完成，移除攻击指令
        Owner.RemoveComponent(ComponentId<AttackCommandComponent>.TId);
    }
}

3. 世界的创建与驱动

using Core.Capability;
using UnityEngine;

public class MyGameWorld : CapabilityWorld
{
    public override void OnInitialize(int maxComponentCount)
    {
        base.OnInitialize(maxComponentCount);
        // 初始化能力系统，分配池容量
        InitCapabilities(maxCapabilityCount: 128, maxTag: 64, estimatedEntityCount: 512);
    }
}

public class GameMain : MonoBehaviour
{
    private MyGameWorld m_World;

    void Start()
    {
        m_World = new MyGameWorld();
        m_World.OnInitialize(1024);

        // 1. 创建敌人
        CEntity enemy = m_World.AddChild();
        enemy.AddComponent<HealthComponent>().HP = 100f;
        enemy.AddComponent<PositionComponent>().WorldPosition = new Vector3(5, 0, 5);
        
        // 2. 创建主角
        CEntity player = m_World.AddChild();
        player.AddComponent<PositionComponent>().WorldPosition = Vector3.zero;
        player.AddComponent<FactionComponent>().FactionColor = Color.blue;
        
        // 为主角绑定能力
        m_World.BindCapability<MovementAndColorCapability>(player);
        m_World.BindCapability<AttackCapability>(player);

        // 3. 下达移动指令
        var moveTarget = player.AddComponent<MoveTargetComponent>();
        moveTarget.TargetPosition = new Vector3(5, 0, 4); // 移动到敌人身边
        moveTarget.Speed = 2f;

        // 4. 下达攻击指令（通常攻击指令会在到达目标后由其他逻辑派发，这里仅做演示）
        var attackCmd = player.AddComponent<AttackCommandComponent>();
        attackCmd.Target = enemy;
        attackCmd.Damage = 150f;
    }

    void Update()
    {
        // 驱动世界运行
        m_World?.OnUpdate(Time.deltaTime, Time.realtimeSinceStartup);
    }
}

三、 这套系统的内部细节

为了做到高性能和去耦，该框架在底层设计了以下巧妙的机制：

1. 极速的组件访问机制

CEntity 内部没有使用缓慢的 Dictionary<Type, Component>，而是使用了基于 ID 的 IndexedObjectArray<CComponent>。ComponentId<T>.TId 在泛型类第一次加载时静态分配整数 ID。这使得 GetComponent 本质上是 O(1) 的数组索引访问，极大提高了运行效率。

2. 响应式的能力激活模型 (CapabilityCollector)

能力 (Capability) 并不是每帧去主动遍历全图实体。在 CapabilityBase.OnInit() 中调用 Filter(...) 时，框架会创建一个 CapabilityCollector。 该收集器会去监听 EntityGroup 的 GroupAdded、GroupRemoved 等事件。只有当关联组件发生增删时，才会将 TryComponentChanged 标记设为 true，从而在 CapabilityRegistry 的主循环中触发一次 ShouldActivate / ShouldDeactivate 的重新评估。状态未变化的帧，不会浪费性能做条件判断。

3. EntityGroup 与 Matcher 的缓存共享

世界 CapabilityWorldBase 维护了一个 Dictionary<EntityMatcher, EntityGroup>。当多个能力关注相同的组件组合时，它们会复用同一个 EntityGroup。当任何实体的组件发生变化时，世界会调用 NotifyComponentChanged，并通过 m_GroupsByComponent 增量更新受影响的特定 Group，避免了全局遍历。

4. 状态互斥与技能阻塞机制 (Tag Block)

除了组件条件，系统还内置了基于 CapabilityBlockComponent 的阻塞机制。在 CapabilityRegistry.UpdateCapabilityArray 的遍历中，如果当前 Capability 携带的 Tag 被该实体的 BlockComponent 标记为阻塞计数大于0，则会直接 continue 跳过当前能力的 Tick 逻辑。这非常适合用来实现例如“眩晕状态下禁止移动能力”、“施法期间禁止攻击能力”等复杂的互斥业务需求。
